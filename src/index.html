Methods

The Disease-Symptom Prediction Chatbot was built using Natural Language Processing (NLP) techniques and a K-Nearest Neighbors (KNN) machine learning model. Data preparation involved curating datasets with symptom descriptions, severity levels, and precautions from CSV and JSON files. User inputs were preprocessed through tokenization, lemmatization, and stopword removal to normalize and refine the data.

The chatbot uses TF-IDF vectorization and cosine similarity to match user-reported symptoms with predefined symptom data. A KNN model, fine-tuned on this dataset, predicts possible diseases and generates comprehensive responses, including descriptions and precautionary measures. To enhance understanding, semantic analysis and part-of-speech tagging were applied to analyze input context accurately.

User interaction is managed via a Flask-based web interface, allowing real-time symptom input. The chatbot refines predictions iteratively by confirming symptoms and prompting for additional inputs. The architecture integrates modular components, enabling efficient preprocessing, symptom matching, and response generation.

While robust, the system's predictions are limited to predefined symptoms and diseases, emphasizing the need for continuous updates to expand coverage. Built using Python libraries like NLTK, SpaCy, and scikit-learn, the chatbot ensures scalability and provides a user-friendly interface to assist in preliminary disease identification.

Limitations

The Disease-Symptom Prediction Chatbot sometimes provides plausible-sounding but inaccurate or incomplete responses. This issue is challenging to resolve because:

  • No Definitive Source of Truth: Thereâ€™s no source of truth during model training, making it difficult to ensure absolute accuracy in predictions.
  • Caution May Lead to Avoiding Correct Answers: Optimizing the chatbot to be cautious may cause it to decline questions that it could answer correctly.
  • Supervised Training Limitations: Supervised training can mislead the model, as the ideal answer depends on symptom-disease relationships, which may not always align with the training data.

The chatbot can be sensitive to small changes in user input phrasing. A slight rewording of a symptom description might lead the chatbot to give an entirely different prediction, resulting in inconsistent or incorrect responses.

The chatbot tends to be verbose, offering overly detailed responses that may not always be necessary. This is due to biases in the training data, where longer responses are preferred, leading the model to prioritize comprehensive-sounding answers.
